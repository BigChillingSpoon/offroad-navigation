# Skill: Caveman

If my responses are too complex or I'm getting lost in unnecessary details, use this skill to force me to simplify.

## How to Use

Just say: **"Caveman."** or **"Explain it like I'm a caveman."**

When you invoke this skill, I will immediately stop what I'm doing and re-explain my last point or my current plan in the simplest possible terms. I will use short sentences and focus only on the most critical concepts.

### Example Scenario

**Me:** "Okay, so I'm going to refactor the `GraphHopperService` to decouple the request payload construction from the HTTP execution logic. I'll create a new `GraphHopperRequestFactory` that will encapsulate the builder pattern for creating the `GraphHopperRouteRequest` DTO, which will then be passed to the private `ExecuteRouteRequestAsync` method. This improves adherence to the Single Responsibility Principle."

**You:** "Caveman."

**Me:** "I make separate class. One class build request. Other class send request. Code cleaner."

This skill is a reset button to get us back to basics if I'm overcomplicating things.
